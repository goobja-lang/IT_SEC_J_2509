효율성 순위 - 코드량이 적고, 딸깍으로 되는 수준
1. Nodejs(typescript) - 단점은 인공지능 안됨
2. FastAPI - 인공지능하기 편해서 이걸로 실습 예정
3. .NETCore(웹, 게임에 많이 사용) - 한국에서는 배척 / 많이 사용 안해봐서 무서움
4. 스프링(자바) - 공공 기관에서 많이 씀
5. PHP - 쇼핑몰 만들 때는 끝내줌


파이썬은 버전 문제가 심각해요.
그래서 어떤 코드는 3.9 에서만 작동
어떤 코드는 3.10 에서만 작동
어떤 코드는 3.11~12 에서만 작동
어떤 놈은 3.13까지 지원

Tensorflow - 구글, 거진 유기함. 근데 문제는 시중에 책은 다 tensorflow로 인공지능 알려줌
 - 얘는 3.7~9 까지만 지원
 - 오류가 많이 발생할 여지가 많음

 Pytorch - Meta, 3.10~3.13 까지만 지원
  - 회사에서 하실 분들은 이걸로 해야함 : 현재 추세임
  - 요즘은 다 Pytorch 로 이사갔어요.
  - 왜나면, 인공지능은 분야가 너무 어렵고 스트레스가 심해서, 개인이
  모든 AI 모데을 다 만들 수 없고, 오픈소스 모델들 갔다 쓰는게 허다해요.
  근데, Pytorch 경험이 없으면 이때 난감해져요.
